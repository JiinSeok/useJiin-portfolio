모두 유닉스 기반 운영체제의 커맨드 라인 인터페이스(CLI)에서 사용할 수 있는 도구입니다.

- `npx`는 패키지를 설치하지 않고도 실행할 수 있는 편리한 도구입니다.
- `npm`과 `yarn`은 로컬 또는 전역으로 설치된 패키지를 사용하는 전통적인 방식입니다.
- `yarn`은 `npm`보다 더 빠르고 효율적인 패키지 관리 기능을 제공합니다.

## npm (Node Package Manager)

- Node.js 환경에서 JavaScript 패키지를 관리하는 기본 도구
- 패키지를 설치, 업데이트, 제거하는 등의 작업을 수행
- 프로젝트 의존성 관리에 사용

## npx (Node Package Execute)

- npm 5.2.0 버전부터 도입된 도구
- 로컬에 설치되지 않은 패키지를 임시로 다운로드하여 실행
- 특정 패키지를 실행할 때 편리

## yarn

- npm과 유사한 JavaScript 패키지 관리 도구
- Facebook에서 개발했으며, npm과 호환되는 기능을 제공
- 빠른 설치 속도와 보다 안정적인 의존성 관리

## 차이점 비교

### 1. 설치 여부

- `npm`은 로컬 또는 전역으로 설치된 패키지를 실행합니다.
- `npx`는 설치 여부와 관계없이 패키지를 실행할 수 있습니다.
- `yarn`은 로컬 또는 전역으로 설치된 패키지를 실행합니다.

### 2. 패키지 다운로드

- `npm`은 패키지를 먼저 다운로드한 뒤 실행합니다.
- `npx`는 패키지를 필요할 때마다 즉시 다운로드하고 실행합니다.
- `yarn`은 패키지를 먼저 다운로드한 뒤 실행합니다.

### 3. 캐싱

- `npm`은 다운로드한 패키지를 캐싱합니다.
- `npx`는 임시 파일로 패키지를 다운로드하고 실행 후 삭제합니다.
- `yarn`은 다운로드한 패키지를 캐싱합니다.

### 4. 편의성

- `npm`은 로컬 또는 전역 설치된 패키지를 사용해야 합니다.
- `npx`는 특정 패키지를 실행하는 데 편리합니다.
- `yarn`은 `npm`과 유사하지만, 더 빠르고 효율적인 패키지 관리 기능을 제공합니다.
