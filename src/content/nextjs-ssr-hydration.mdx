## Server-Side Rendering 과정

1. 사용자가 Next.js 웹 페이지를 요청하면 **Next.js 서버에서 페이지를 생성**합니다.
2. 서버는 사용자가 요청한 페이지의 컴포넌트를 렌더링하여 HTML 파일을 생성합니다.
3. 서버에서 미리 렌더링된 HTML 페이지는 클라이언트로 전송됩니다.
4. 사용자의 브라우저에 도착하면, "hydration" 과정이 실행됩니다.

## Hydration 과정

Hydration은 클라이언트에서 서버에서 생성된 HTML 페이지를 활성화하는 과정입니다:

1. 브라우저에서 JavaScript 코드가 실행되면서 서버에서 보낸 HTML 페이지를 인식합니다.
2. 브라우저의 React 인스턴스는 서버에서 생성된 HTML의 DOM 구조와 일치하도록 업데이트(동기화)됩니다.
   - 이 과정을 통해 사용자는 즉시 페이지를 볼 수 있게 됩니다.
3. JavaScript 코드가 완전히 로드되면 페이지가 완전히 동적으로 작동하게 됩니다(이벤트 핸들러 등).

이러한 SSR과 Hydration 과정을 통해 Next.js는 빠른 초기 로딩 속도와 SEO 최적화, 그리고 동적인 사용자 경험을 모두 제공할 수 있습니다.
