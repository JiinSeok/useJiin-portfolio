TypeScript에서 `type`과 `interface`는 모두 객체의 형태를 정의하는 데 사용되지만, 몇 가지 중요한 차이점이 있습니다.

## 1. 선언적 확장

`interface`는 같은 이름으로 중복 선언을 함으로써 확장이 가능하지만, `type`의 경우 에러가 발생합니다.

```typescript
// Interface - 선언 병합 가능
interface User {
  name: string
}

interface User {
  age: number
}

// 결과적으로 User는 { name: string; age: number; }

// Type - 중복 선언 시 에러 발생
type Animal = {
  name: string
}

// Error: Duplicate identifier 'Animal'
type Animal = {
  age: number
}
```

## 2. `extends`, `&`, `|`

`interface`는 `extends` 키워드를 통해 확장할 수 있습니다. 비슷한 기능을 `type`에서 구현하려면 intersection 타입인 `&`를 통해 사용할 수 있습니다.

```typescript
// Interface - extends 사용
interface Person {
  name: string
}

interface Employee extends Person {
  employeeId: number
}

// Type - intersection 타입(&) 사용
type Person = {
  name: string
}

type Employee = Person & {
  employeeId: number
}
```

반면, union 타입은 `interface`에서 비슷하게 구현할 수 없습니다.

```typescript
// Type - union 타입(|) 사용 가능
type ID = number | string

// Interface - union 타입 직접 구현 불가능
// 아래와 같은 방식으로는 구현할 수 없음
// interface ID = number | string; // 에러 발생
```

## 3. 기타 차이점

### 계산된 프로퍼티

`type`은 계산된 프로퍼티를 지원하지만, `interface`는 지원하지 않습니다.

```typescript
type Keys = 'firstName' | 'lastName'

// Type - 계산된 프로퍼티 지원
type NameMap = {
  [K in Keys]: string
}

// Interface - 계산된 프로퍼티 미지원
// interface NameMap {
//   [K in Keys]: string; // 에러 발생
// }
```

### 튜플과 배열

`type`은 튜플과 배열을 더 간결하게 표현할 수 있습니다.

```typescript
// Type - 튜플 표현
type Point = [number, number]

// Interface - 튜플 표현 (더 복잡함)
interface PointInterface {
  0: number
  1: number
  length: 2
}
```

## 어떤 것을 사용해야 할까?

일반적으로 다음과 같은 기준으로 선택할 수 있습니다:

- 객체의 형태를 정의하는 경우 → `interface` 사용 (확장성이 좋음)
- Union 타입, 튜플, 기타 복잡한 타입이 필요한 경우 → `type` 사용
- 라이브러리나 API를 개발하는 경우 → `interface` 사용 (사용자가 확장할 수 있음)
- 내부 구현에 사용되는 타입 → `type` 사용 (더 유연함)

TypeScript 팀은 일반적으로 가능한 한 `interface`를 사용하고, 필요한 경우에만 `type`을 사용하는 것을 권장합니다.
